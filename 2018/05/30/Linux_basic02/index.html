安全的远程连接+远程拷贝+文件传输

  1. ssh 安全远程连接
    ssh 用户名称@公网地址
  • 退出远程连接 exit / logout
  1. scp 安全远程文件拷贝
  1. sftp 安全文件传输（file transfer protocol)
    • lls 查看本地文件
    • lcd 本地目录跳转
    • lpwd 查看本地目录（加l代表在本地操作，不加l代表在远端操作）
    • put - 上传文件
      • put -r 上传文件夹
    • get - 下载
    • bye / quit - 退出
    • help - 查看帮助(不记得了就在sftp下查)

  1. ping - 检查网络可达性 - ICMP(Internet
    Control Management Protocol) - 网络层

    • ttl - time to live (避免无效的网络请求影响网络速度)

进程相关操作

  1. ifconfig / ip addr 查看本机地址和以太网地址
    • ifconfig lo 查看localhost
    • ifconfig eth0 查看以太网卡
  2. netstat 查看进程网络状态(-nettat -nap |grep 进程或端口号)
    • nap(n以数字形式显示,a显示全部,p 查看进程信息)
    • kill 强制结束进程
  3. systemctr
    • systemctr start nginx 启动nginx
    • systemctr stop nginx 关闭nginx
    • systemctr restart nginx 重启nginx
    • systemctr status nginx 查看nginx运行状态
    • systemctr enable nginx 开机自启nginx
    • systemctr disable nginx 取消开机自启nginx
  4. 后台运行程序(&)
    (前台运行会影响交互式环境 ctr+c 停止运行)

  1. 查看进程
  • 查看redis是否在运行(ps)
  • jobs 查看后台运行或停止的进程
  • netstat -nap | grep 6379

  1. 将后台程序放在前台运行(fg)

  2. 将前台运行程序放到后台运行(bg)
    先 ctr + z 再 bg

  3. top 任务管理器(查看进程占用cpu、内存等资源情况)

  • 退出 q

使用Git实施版本控制

1. 版本控制历史

  • 1990s - CVS / VSS - 锁定模式
  • 2000 - Subversion（SVN）- 合并模式
    必须有版本控制的中央服务器才能使用版本控制 - 中央集权型的版本控制系统
  • 分布式版本控制系统 - 2005 - Git
  • 目前最火 Git / Mercury

2. Git 命令

创建仓库+存放文件
  • git init - 将文件夹初始化为Git仓库

  • git add - 将工作区的指定文件放入暂存区

  • git status - 查看工作区和暂存区的状态

  • git commit -m ‘提交的原因’ - 将暂存区的内容添加到仓库

==注意==：第一次使用时，需要输入用户名和邮箱

  • git log - 查看提交日志（历史提交记录）
误删除如何恢复

  • 情况一：文件在工作区被删除，但是暂存区未删除

    • git checkout – - 将暂存区的文件恢复到工作区

  • 情况二：暂存区的也提交删除了

    • git reset –hard HEAD^ - 回到上一个版本
    • git reset –hard - 回到指定的版本

  • 情况三：恢复时，回版本回错了
    • git reflog - 查看历史和未来版本
